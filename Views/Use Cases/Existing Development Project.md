This use case is as a developer for a GitHub repo or repos. 1 or more Spider Monkeys will code and test moving the development forward to the projects vision.


## Example workflow

User set up the project vision with repos and goals.

Spider Monkey assigned setups development environment

Project Manager sub-brain examines current state of the code base
Project Manager decides on first n steps (<5) that move towards the vision

Create worktree and branches as required

A Code manager, Sub-brain manages a AI Code CLI is asked to plan out the implement of a step and starts development.
A test sub-brain manages a test plan for the step, and waits to run it

Code Manager monitors and advises, answering CLI question as required. When the step is completed, test sub-brain is informed to do a test and passes results back to Code manager.

When ready a PR is created and can either if a PR review project is attached to this repo wait for ti to review or merge, or runs its own PR review system.

Project Manager decides on another step towards the goal if any and then repeats development until the Vision is complete.

The primary brain of the Spider Monkey keeps the user informed and handles manual testing, deploy etc. 

The goal is independent development toward the vision with the User informed and in the loop for major items.
